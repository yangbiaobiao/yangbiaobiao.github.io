面向对象

首先关于构造函数，这里需要先提一下面向对象，

面向对象他是一种思维方式，不关注他的内部结构只在乎它的功能，而且万事万物皆对象！

（没对象的赶快了，，，，）

构造函数

第一种方式 构造函数 工厂模式

function dog(name,sex){
    //在构造函数内部 建立一个对象
    var obj={}    
    //函数属性
    obj.name=name;
    obj.sex=sex;
    //方法：
    obj.bark:function(){
        alert("汪汪~~")
    }
    //返回这个对象
    return obj
}

//实例化 在外面调用属性和方法 
var erha = dog("哈士奇"，“雄性”)
//调用属性 
console.log(erha.name , erha.sex ) // 哈士奇 雄性
//调用属性
console.log(erha.bark()) // 汪汪~~

第二种方式 构造函数

function dog(name,sex){
    //属性
    this.name=name;
    this.sex=sex;
    //方法
    this.bark:function(){
        alert("汪汪~~")
    }    
}
//实例化 
var erha = new dog("哈士奇"，“雄性”)    
//调用属性
console.log(erha.name , erha.sex ) // 哈士奇 雄性
//调用属性
console.log(erha.bark()) // 汪汪~~

但是这两种方式有点小瑕疵 就是我们每次在实例化的时候每次都会new出方法，在调用方法的时候，调用一次，创建一次对象

new出它的方法，调用一次，创建一次对象，所以这在使用时多多少少会有点浪费资源，所以我们经常的写法应该是将属性和

方法分开写 并且把方法写在它的原型链上（下面会解释，，），这样相比较就节省资源了，不用每次都new出方法

具体的写法：

function dog(name,sex){
    this.name=name,
    this.sex=sex
}
dog.prototype = {
    bark:function(){
        alert("汪汪~~")
    }
}
//实例化
var erha = new dog ("哈士奇"，“雄性”)
//调用它的属性
console.log(erha.name , erha.sex ) // 哈士奇 雄性
//调用属性
console.log(erha.bark()) // 汪汪~~

这里为什么这样写就可以解决上面的问题哪？

原因就在于对象的原型属性 prototype 属性

这里的prototype属性 说起来感觉比较麻烦，而这里就和大家说一个比较好理解的方式，就是把这个prototype属性以
我们的class属性来理解，就比较好理解了，class名可以多个标签来使用，但是style样式是不是只能在行间使用而且只
能单独使用呀！
